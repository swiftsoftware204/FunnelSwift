Plans

Manage subscription tiers and Kinetic Card feature toggles at /admin/plans. Each plan controls which features users can access.

How Plans Work

When a user signs up, the form sends a plan field that routes them to the right plan:

  • funnelswift.net signup — no plan field → defaults to the free plan
  • funnelswift.net/kinetic signup — sends kinetic_free → maps to the kinetic_free plan

Kinetic Cards

Create bio-link pages and mini landing pages using the Kinetic Cards system. Each card gets a unique slug at funnelswift.net/k/[slug].

New in v2.0: Kinetic Cards replace static link-in-bio pages with dynamic, SSR-rendered layouts. Choose between Bio-Link (Linktree-style) or Mini-Page (section-based landing page).

Card Types

TypeUse Case
Bio-LinkAvatar + bio + pill buttons + social icons. Like Linktree but self-hosted.
Mini-PageSection-based landing page with hero, features grid, lead capture form.

Layout Blocks (Mini-Page)

Mini-Pages render layout_blocks as a sequence of sections.

BlockDescription
heroHeadline, subheadline, CTA button. Custom background color and optional video.
featuresGrid of feature cards with icon, title, description.
contentRich text or image + text section.
ctaStandalone call-to-action block with button and link.
galleryImage grid display.
videoEmbedded video (YouTube/Vimeo provider).
formLead capture form that submits directly to FunnelSwift.

Pill Buttons (Bio-Link)

Each bio-link card supports up to 20 pill buttons with action types:

  • link — Navigate to a destination URL
  • form — Open an inline lead capture form
  • call — Initiate a phone call (tel:)
  • sms — Send an SMS (sms:)

Lead Source Tracking

Each Kinetic Card can define source parameters that map URL query params to tags. Example: ?utm_source=instagram auto-tags leads with the matching tag from the card's source config.

QR Codes

Each Kinetic Card can generate a QR code linking to its slug URL. Use for print materials, business cards, or signage.

Pro tip: Combine Kinetic Cards with Campaigns (see below) to track which cards drive the most conversions.

Campaigns

Campaigns bridge FunnelSwift leads with IncentiveSwift reward programs. Create targeted campaigns, assign them to Kinetic Cards, and track performance.

Requires: A valid IncentiveSwift API key linked to your FunnelSwift tenant.

How Campaigns Work

  1. Create in IncentiveSwift

    Set up the campaign with rewards, eligibility rules, and tracking parameters.

  2. Link to FunnelSwift

    FunnelSwift proxies campaign data from IncentiveSwift using your tenant's API key. Campaigns appear in the admin dashboard automatically.

  3. Assign to Kinetic Cards

    Tag campaign IDs on Kinetic Card buttons to associate leads with specific campaigns.

  4. Track Performance

    Campaign analytics show lead attribution, conversion rates, and reward redemption stats.

Webhooks

Configure outbound webhooks to push lead events and system notifications to external services in real time.

New in v2.0: Webhook dispatch with automatic retry and exponential backoff — no data loss on transient failures.

Event Types

EventTriggered When
lead.createdA new lead is created via any channel (form, SMS, API).
lead.updatedLead record is modified (status change, tag assignment).
lead.convertedLead status changes to Converted.
tenant.updatedTenant settings or branding changes.
affiliate.createdNew affiliate partner added.
plan.updatedPlan configuration modified.

Creating a Webhook

  1. Go to Admin → Webhooks
  2. Click + Add Webhook
  3. Name it

    e.g. "Slack Lead Notifications"

  4. Enter URL

    The endpoint that will receive POST payloads.

  5. Select events

    Comma-separated: lead.created, lead.converted

  6. Save

    Webhook is active immediately.

Payload Format

{
  "event": "lead.created",
  "timestamp": "2026-07-17T07:00:00Z",
  "data": {
    "id": "uuid",
    "name": "John Doe",
    "email": "john@example.com",
    "source": "Web Form",
    "status": "new"
  }
}

Retry Behavior

Failed deliveries retry with exponential backoff: 10s, 30s, 90s, 270s, then final. If all retries fail, the event is logged for manual review.

Testing

Use the Test button on any webhook to send a sample payload. Check the target endpoint's logs to verify delivery.

Web-to-Lead

Capture leads from any external website by embedding a small JavaScript snippet. When a visitor submits a form on your site, the lead is automatically created in your FunnelSwift dashboard.

No API key needed! The embed snippet uses a built-in public key. Just copy and paste — no editing required.

Creating a Config

  1. Go to Web-to-Lead

    Click Web-to-Lead in the sidebar.

  2. Click + New Config
  3. Name it

    e.g. "Website Contact Form"

  4. Set default source

    e.g. "Web Form" — this labels where the lead came from.

  5. Auto-tag leads

    Select tags to apply automatically when a lead is captured from this form.

  6. Save

    Your config is created. Click the link icon to get the embed code.

Getting the Embed Code

Click the link icon (🔗) next to any config to open the embed modal. You get two options:

H

Option A: In the <head>

Paste this code into the <head> section of your website. It uses the defer attribute so it loads in the background without slowing your page down.

B

Option B: Before </body>

Paste this just before the closing </body> tag. This guarantees all your form elements are fully loaded before the script starts watching for submissions.

Where to Paste It

PlatformInstructions
WordPressInstall WPCode plugin → Add New → "Header & Footer" → paste in Header or Footer.
WebflowSite Settings → Custom Code → paste in "Head Code" or "Footer Code".
SquarespaceSettings → Advanced → Code Injection → paste in Header or Footer.
WixSettings → Custom Code → + Add Custom Code → paste script → select "All Pages".
Custom HTMLPaste Option A in the <head> or Option B just before </body>. For a single-page form, paste below your <form> tag.
ClickFunnelsSettings → Tracking Code → paste in Body section.

How It Works

1

You Paste

Copy the script from FunnelSwift and paste it onto your external website (WordPress, Webflow, Squarespace, etc.).

2

Visitor Submits

When someone fills out the form on your site and hits submit, the script intercepts the data (Name, Email, Phone).

3

Lead Created

The script sends the form data to FunnelSwift. A new lead appears instantly in your dashboard with the tag you selected.

Note: The script does NOT prevent the normal form submission. Your existing form will still work as before — the lead capture happens in the background.

City Requests

The City Request system lets visitors suggest new cities (or products/niches) directly from the directory homepage. Manage all requests from one network-wide admin dashboard.

How It Works

  1. Visitor submits

    At the bottom of the homepage, visitors enter a city and state. Submissions are stored in a central database.

  2. Deduplication

    If 3 people request "Queens, New York", the count goes to 3. No duplicate entries.

  3. Admin reviews

    Open the admin dashboard to see all requests sorted by popularity.

  4. Add to directory

    When you add a requested city, click "Add to Directory" and it disappears from the request list.

Admin Dashboard

Accessible at https://zaarhub.com/admin/requests (or your directory’s /admin/requests path).

FeatureHow To
View requestsDashboard opens to the Requests tab. Cities sorted by request count (highest first).
Remove a cityClick "Add to Directory" — it’s removed from the list instantly.
Edit form textSwitch to the Form Settings tab. Change headline, description, button text, placeholders, and success messages.

Customizing the Form

The city request form is fully customizable from the admin settings. Use it for any niche:

ScenarioHeadlinePlaceholder

Service directory"Need a service we don’t have?""Service type"
Note: The city request system is network-wide. One database, one admin dashboard — even if you have multiple directories. New cities can be added to any directory in the network.
